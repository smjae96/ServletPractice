POST 방식으로 요청 후 응답 페이지 보기

<> 특징1. URL의 body 영역에 데이터를 포함시켜 요청
=> 사용자가 입력한 값들이 url에 노출되지 않음 => 보안 상 안전함
=> 그러므로, 로그인이나 회원가입 같은 경우 POST방식이 적합

특징2. body영역에는 전송 길이의 제한이 없음
=> 게시글 작성 시 POST 방식이 적합

특징3. 즐겨찾기(북마크)는 가능하나, 데이터가 url에 노출되지 않아 기존 화면과 다를 수 있음
=> 검색 기능 같은 경우 POST 방식이 부적합

특징4. 타임아웃(timeout)이 존재하기 때문에 페이지 요청에 기다리는 시간 있음
